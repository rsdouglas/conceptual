# ConceptDefinition

**Type:** entity
**Bounded Context:** Source Code Analysis and Conceptual Modeling
**Aggregate Root:** No

---

## 1. Definition

**Short Description:**

Defines a single concept within the conceptual model, including its type, metadata, and structure.

**Ubiquitous Language:**

ConceptDefinition refers specifically to the domain entity that encapsulates the definition of a concept extracted from source code, including its descriptive metadata and structural details. Synonyms like 'Concept Model' or 'Concept Entity' are discouraged to avoid ambiguity.

---

## 2. Structure

### Fields

- `shortDescription: string` — A concise, plain-English summary of the concept.
- `ubiquitousLanguage: string` — Optional detailed explanation of the term's precise meaning and acceptable synonyms.
- `metadata: ConceptMetadata` — Metadata describing the concept's name, type, bounded context, aggregate root status, and criticality.
- `structure.fields: ConceptStructureField[]` — Domain-relevant properties that define the concept's internal structure.
- `structure.relationships: ConceptRelationship[]` — Descriptions of meaningful domain relationships to other concepts.
- `lifecycle: ConceptLifecycle` — Optional lifecycle states and valid state transitions for the concept.
- `invariants: ConceptInvariant[]` — Non-negotiable business rules that must always hold true for the concept.
- `commands: ConceptCommand[]` — Actions that modify or interact with the concept.
- `events: ConceptEvent[]` — Domain events emitted when significant changes occur to the concept.
- `implementation: ImplementationLink[]` — References to code files, symbols, or URLs implementing or documenting the concept.

### Relationships

- May be referenced by other concepts within the conceptual model.
- Is part of the overall conceptual model generated by the Concept Generation Service.
